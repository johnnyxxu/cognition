Documents API

This is a basic "hello world"-style RESTful API that can be used to store documents. Each doc has a name and content.

The route can be found at /docs. Note that it requires authorization and is secured with TLS.

GET /docs

Returns all documents.

GET /docs/:name

Fetch a document by :name.

POST /docs

Create a new document. Provide a json request like:


    {
      "name": "notes",
      "content": "today was a good day"
    }

name is required, must be unique, and can only contain letters, numbers, dashes, and underscores.

PUT /docs/:name

Update a document by :name, providing a json request like the one shown above.

DELETE /docs/:name

Delete a document by :name.
